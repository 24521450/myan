Anki Card CSS — vùng 2

The CSS between the ANKI CARD STYLES markers in <head> above. This block is the contract: it must be 1:1 with design/EAVM/styling.txt, modulo rules marked with the preview-only marker comment. Drift enforced by python -m tools.check_design_sync.

Sync rules:
• All selectors & properties between the two markers must match EAVM/styling.txt exactly.
• Rules marked with the preview-only marker (see _design_sync.py for exact token) are skipped on both sides (excluded from drift check, not baked into .apkg).
• @-rules (@import, @keyframes, @media) are ignored by the parser — they live in styling.txt but don't count toward drift.
• Selector names are immutable contracts — renaming breaks every template that references them.

Card design rules (enforced at build stage)
① Sense Cap
Max 3 defs/card. Nếu raw note có > 3 senses, giữ 3 phổ biến nhất (priority: sensenum_local asc → example count desc). Raw notes giữ nguyên — filter ở build, không phải scrape. Helper: tools/_filter_top3_defs.py.
② Card Identity
1 CEFR = 1 card. Multi-POS (vd absent = adj/verb/prep) → 1 card với POS chips list tất cả. Cùng word khác CEFR → 2 cards riêng. CEFR trống → cefr-badge-UNCLASSIFIED.
